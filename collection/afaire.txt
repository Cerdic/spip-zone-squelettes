Projet de squelettes pour présenter une collection d'objets physiques.

Contraintes
-idéalement, adapté à n'importe quel type d'objet ->pas de champs extra, mais des mot-clefs, nombre illimités -> c'est le cas
- en plugin (fait)
- permettre de faire une recherche en combinant plusieurs critères (ex : taille, sexe (pour les schtroumpfs !) (fait, mais à améliorer avec le futur plugin de _fil_)
- configurable via CFG, notamment au niveau des couleurs (un début)
- reprendre la terminologie de zest and zen, avec les mêmes noms de classs et d'id, pour pouvoir facilement reprendre un thème.Mais j'avoue ne pas avoir trop bien suivis le fonctionnement. (la convention est pas encore fixé, j'ai repris la dist, et il faudrait faire aussi avec le système d'inclusion profond)
- portfollio avec ThickBox1 (fait)
- crayons ! (fait)
- ajax ? (fait)

Organisation logique :
- rubrique unique
- critères sont des mot clefs
- chaque objet est un article. 
*Titre->nom de l'objet ; 
*texte -> descriptif de l'objet ;
*date -> date d'ajout à la collection ; -> pas encore mis dans les squel
*date antérieur -> date de création de l'objet ;

- pas de logo, mais des images jointes (pour avoir plusieurs image pas objet). Une image principale servant pour le portfolio de tout les objets. (fait)
